GuardDuty found that there was an unauthorized access attempted via the SSH Brute Force attack. It collected the information on the resources affected. It also collected the information on which port the attack happen, which EC2 instance, security groups etc. GuardDuty collects the information from the following sources: VPC Flow Logs, AWS CloudTrail management event logs, Cloudtrail S3 data event logs, and DNS logs. It looks for unusual activity on the servers. It also uses AI to further recognize the possible issue. 

In case of "our" attack, GuardDuty is constantly watching selected ports. SSH uses port 22. If it sees unusual activity on port 22 as well as multiple attempts to log in. It recognizes this as the brute force attack.
